# Sub-projects 0 + A + B — Podman foundation, data plane, observability

**Date:** 2026-06-25
**Scope:** First slice of the QA/AQA infrastructure. Single bootstrap script that installs Podman, provisions a podman-machine, and stands up 9 services across data plane and observability.
**Non-goals:** No application services yet (model-gateway, RAG, agents, gateway). Those land in C and D.

## Global invariants (apply to every sub-project)

- **Open-source first**: no SSPL/proprietary components. Valkey replaces Redis. Neo4j Community (GPLv3). Grafana OSS (AGPLv3). All others permissive (MIT/Apache/BSD/PostgreSQL).
- **One file, one command**: `infra/infra.sh` is the only edit surface. `./infra/infra.sh` is the only command. Idempotent.
- **Dev → server → k8s portability**: compose is the source of truth in this file; `dist/compose.yaml` is generated by the script (or embedded directly). Conversion to systemd Quadlets / k8s Pod YAML comes in later sub-projects via `dist/` outputs.
- **LangChain + LangGraph** reserved for Sub-project D agent-workers (not relevant here).

### Network

Single bridge network `qa-aqa`. All services reach each other by container name (e.g. `postgres:5432`). Only ports listed above are published to host.

### Secrets

`infra/.env` (generated on first run from defaults; never overwritten). Includes Postgres password, Neo4j password, Grafana admin password. `infra/.env.example` committed.

## `infra/infra.sh` structure

Single bash script. 8 numbered sections, all clearly labelled. Re-runnable; each section is a function gated by an idempotency check.

1. **Variables** — versions, ports, machine size (CPU 6, RAM 12GB, disk 100GB). Top of file. ONE place to bump versions.
2. **Prereq check + install** — `brew install podman podman-compose just yq jq`. Skips if installed.
3. **Podman machine** — `podman machine init --cpus 6 --memory 12288 --disk-size 100` if absent; `podman machine start` if not running.
4. **Generate dist/compose.yaml** — heredoc → file. Regenerated every run from the script's current state.
5. **Generate dist/otel-config.yaml, prometheus.yaml, grafana provisioning** — heredocs → files.
6. **Generate infra/.env** if absent. Never overwrites.
7. **`podman-compose -f dist/compose.yaml up -d`** — idempotent up.
8. **Wait + smoke** — 8 smoke tests above, in dependency order. Fail loudly with diagnostic on any miss.

## Risks & mitigations

| Risk | Mitigation |
|---|---|
| First `podman machine init` downloads ~2 GB image | Script announces this clearly and exits cleanly on interruption |
| Apple Silicon emulation tax on x86 images | Use ARM64-native tags where available; OpenSearch + Neo4j + Postgres all publish ARM64 |
| OpenSearch JVM grabs more heap than budget | Set `OPENSEARCH_JAVA_OPTS=-Xms1g -Xmx1g` explicitly |
| `podman-compose` quirks vs Docker Compose v3 | Pin podman-compose version; document any rewrites in compose YAML comments |
| Re-run wipes user secrets | `.env` write is gated on absence; never overwrites |
